# Forces of Progress Canvas

A printable / digital canvas for mapping the four forces that determine whether a customer switches to a new solution.

```
                  PRO-SWITCH FORCES                       ANTI-SWITCH FORCES
              ----------------------------          ----------------------------
              |                          |          |                          |
              |  PUSH                    |          |  ANXIETY                 |
              |  (of the situation)      |          |  (of the new solution)   |
              |                          |          |                          |
              |  What's bad about the    |          |  What worries customers  |
              |  current state?          |          |  about switching?        |
              |                          |          |                          |
              |  - [pain 1]              |          |  - [anxiety 1]           |
              |  - [pain 2]              |          |  - [anxiety 2]           |
              |  - [pain 3]              |          |  - [anxiety 3]           |
              |                          |          |                          |
              ----------------------------          ----------------------------

              ----------------------------          ----------------------------
              |                          |          |                          |
              |  PULL                    |          |  HABIT                   |
              |  (of the new solution)   |          |  (of the present)        |
              |                          |          |                          |
              |  What's attractive       |          |  What ties customers     |
              |  about the alternative?  |          |  to the current way?     |
              |                          |          |                          |
              |  - [appeal 1]            |          |  - [tie 1]               |
              |  - [appeal 2]            |          |  - [tie 2]               |
              |  - [appeal 3]            |          |  - [tie 3]               |
              |                          |          |                          |
              ----------------------------          ----------------------------

      A switch happens when:  PUSH + PULL  >  ANXIETY + HABIT
```

## Force Balance Analysis

Sum each quadrant's strength scores:

| Force | Total Score | Direction |
|-------|-------------|-----------|
| Push | [sum] | Pro-switch |
| Pull | [sum] | Pro-switch |
| Anxiety | [sum] | Anti-switch |
| Habit | [sum] | Anti-switch |

**Switch likelihood:** `(Push + Pull) - (Anxiety + Habit)`

- If positive: switch likely (current customers will hire your product)
- If negative: switch unlikely (your product's pull is overpowered by anxiety + habit)
- If near zero: borderline; specific moves needed

---

## Strategic Moves by Force

For the **dominant anti-switch force** (whichever of Anxiety/Habit has the higher score):

### If Anxiety dominates:

- [ ] Free trial or money-back guarantee
- [ ] Migration assistance / white-glove onboarding
- [ ] Detailed case studies with named customers in similar situations
- [ ] Security / compliance certifications
- [ ] Side-by-side comparison ("run both for 30 days")

### If Habit dominates:

- [ ] Import tool from incumbent solution
- [ ] Familiar UX patterns that mirror the old way
- [ ] Preserve existing workflows during transition
- [ ] Allow dual-running for a defined period
- [ ] Provide muscle-memory bridges (keyboard shortcuts, export formats)

### Sharpen PUSH:

- [ ] Marketing copy that names specific pains
- [ ] Case studies that quantify the cost of inaction
- [ ] Onboarding moments that contrast new vs. old

### Refine PULL:

- [ ] Specific differentiation against *named* alternatives (not generic benefits)
- [ ] Demo flows that highlight the underserved outcomes
- [ ] Proof points tied to ODI top opportunities (high importance, low satisfaction)

## Output Hand-off

The forces map feeds into:

- `discovery/identify-assumptions/` -- forces with weak evidence become assumptions
- `discovery/brainstorm-experiments/` -- moves to reduce anxiety/habit become experiment hypotheses
- `execution/create-prd/` -- forces inform Sections 3 (Background) and 6 (Value Propositions)
- `marketing/` -- push/pull messaging directly informs go-to-market copy
- `business-growth/` -- anxiety-reducing moves often live with sales (onboarding, migration)
